# 一维数据的格式化和处理
## 数据组织的维度介绍
- 一维数据和二维数据；集合列表等就是一维的数据，类似于表格就是二维数据
- 多维数据：如果表格加上时间之后又有不同时间的表格数据，这就是多维数据
- 数据的操作周期一般分为数据的储存，表示，操作
- 在数据的储存期间我们关注数据的储存类型，在在数据的操作过程我们关注数据的操作方式，在数据的表示过程我们关注数据的类型

## 一维数据的表示
- 一维数据表示就是如何使用程序来表示一维数据
- 如果一维数据之间是无序的，可以使用集合
- 如果一维数据之间是有序的，可以使用列表

看起来很简单，其实真的很简单

## 一维数据的存储
- 方式一：使用空格分隔（‘ ’），中间不能换行，都写在一行 缺点:数据之间不可以有分隔
- 方式二：使用逗号分隔（‘，’），注意必须是使用英文半角逗号，缺点同样是数据中不能有逗号
- 方式三：其他符号分隔（‘自己根据数据特点确定使用的分隔符号’），确定数据中不可能有某一种字符就使用该字符分隔。  缺点就是通用性差

# 二维数据的格式化处理
## 二维数据的表示
- 使用二维列表来表示（列表里面还有列表，每一个列表就是一行数据，若干行数据构成了二维数据）

## CSV格式与二维数据的存储
- CSV格式就是使用逗号分隔的数据，每一个数据之间都使用逗号分隔（没有空格
- CSV格式中如果储存的数据有缺失，逗号依然是需要保留的，这是规则
- 如果储存的数据中有逗号一般会在数据两侧加引号

## 二维数据的存储
- 二维数据按照行存和列存都可以
- 一般的索引习惯是先行后列，就是先检索数据在哪一行然后看数据在哪一列（一般喜欢按行存）
