## python的异常以及处理方案
- 异常：Bug，会中断程序运行

## 异常处理
1. 不做处理：程序遇到Bug，中断运行
2. 捕获异常：按照处理方式处理异常，程序继续执行
3. except是不限次数的，可以有很多个except


# 捕获异常
```python
try:
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    可能出现异常的代码
    ........
except [异常类型 as 变量名]：#加【】可以捕获指定类型异常，加变量名可以获取异常信息，如果对与获取异常信息不关注，可以不加变量名
    出现异常的处理方案
finally
    不管有没有异常，最终都会执行的部
```
# 异常的传递
- 如果遇到函数的嵌套调用，如果最外层函数抛出异常，异常会层层上报，最终抛给最内层函数，最终异常如果未被处理，则程序崩溃
- 处理异常，最合适位置是在主程序中，捕获所有异常（定义一个异常类，使用面向对象统一进行处理）
