# git的学习和GitHub学习




- 提交时候一定要先放到暂存区，然后提交，然后推送
## git常用命令
1. git config --global user.name chx  (设置用户名)
2. git config --global uesr.email xiaohuanshuijiao@qq.com   (设置用户邮箱，可以是假的)---可以在C盘用户目录下查看（.gitconfig文件）

3. 这个用户签名和未来在GitHub托管代码没有任何关系

### 1.初始化本地库
- git init  
- （初始化本地库，让git获取文件管理权）
### 2.查看本地库的状态
- git status  
- （查看本地库的状态）
### 3.新增文件
- vim hello.txt 
- 新增文件，编辑文件时点击a或者i进入编辑模式，esc可以退出，yy是复制p是粘贴，使用 :wq 来保存文件
- cat hello.txt --------------可以查看文件内容
- tail -n 1 hello.txt----------查看最后一行
### 4.把本地文件添加到暂存区（追踪文件）
- git add 'file'
- 文件名变成绿色，说明git成功追踪到文件
### 5.删除暂存区的文件
- git rm --cached 'file'
- 只是删除了暂存区的文件，工作区任然有的
### 6.提交本地库
- git commit -m"日志信息" 'file'
- 日志信息是支持中文的，提交结束之后会显示一个7个字符的信息，那个就是版本号
### 7.查看文件的历史版本
- git reflog    查看版本信息
- git log     查看详细日志，可以看到是谁什么时候提交的版本，还有完整的版本号（可以看到之前的用户签名）
### 8.回到之前的版本
- git reset --hard 版本号
- 直接变化，不经过暂存区
## git的分支操作
1. 什么是分支?
- 版本控制过程中，需要同时推进多个任务，在开发自己的分支的时候不会影响主线分支的运行，自己开发的那个分支开发坏了也没事
- 分支开发好以后可以合并
### 1.创建分支
- git branch 分支名
### 2.查看分支
- git branch -v
### 3.修改和切换分支
- git checkout 分支名--查看分支的时候分支名变绿说明已经切换到当前分支
- 注意：在分支上修改文件结束之后的操作依然是上传暂存区然后提交，可以在分支进行查看提交版本等等操作
### 4.合并分支
- git merge 分支名2
- 注意：如果进行合并操作是指针指的是分支名1上，使用git merge 分支名2的命令，是把分支名2合并到分支名1上
### 5.处理冲突
- 合并的时候可能会存在冲突
- 冲突产生原因：两个分支在  **同一个文件的同一个位置**  有两套完全不同的修改，git无法替代我们决定使用哪个

案例如下：
![合并冲突案例](https://raw.githubusercontent.com/cheng-energy/energy-python/refs/heads/main/images/%E5%B1%8F%E5%B9%95%E6%88%AA%E5%9B%BE%202026-04-03%20214039.png)
- 之后分支状态会改变（master|MERGING）
- 之后需要手动合并

### 6.手动修改之后的操作注意
- 手动修改之后需要提交到暂存区
- **提交的时候是不可以带文件名的**
- 合并分支并不会修改分支内容，即分支2的内容不会被修改
### 7.分支的重命名
- 强制命名-->git branch -M 旧名 新名
- 安全命名-->git branch -m 旧名 新名
- 如果不写旧名会默认当前分支，注意区分大小写

## git的团队协作机制
- 要用到代码托管中心（团队内协作和跨团队协作）

- 一，团队内协作
1. 本地库通过push推送到代码托管中心（远程库）
2. 另一个人clone克隆复制到自己的本地库
3. 另一个人修改之后push推送到代码托管中心（远程库）--push需要一定权限，要第一个人把另一个人加入到团队中
4. 第一个人就可以pull拉取另一个人修改的内容，这样协作结束

- 二，跨团队协作
1. 团队外的人1通过先fork复制要修改的别人的文件到自己的代码托管中心
2. 然后团队外人1通过clone从代码托管中心克隆文件到本地
3. 团队外人1修改结束之后push到自己的代码托管中心
4. 然后团队外人1要向审核者发送 Pull request来申请权限
5. 审核通过之后团队内人可以merge过来修改后文件，然后可以pull拉取修改后文件到自己的本地库


## GitHub的操作（推送代码到GitHub）
### 1.为自己的远程仓库创建别名
- git remote -v 查看当前所有远程地址的别名
- git remote add 别名 远程地址

### 2.本地库代码推送
- git push 别名或者链接 分支
- 推送时最小单位是分支，注意说明要推送的分支是哪个
### 3.拉取远程库到本地库
- 在网页端修改代码导致和本地库代码不一致可以拉取来同步代码
- git pull 别名或者链接 分支
- 拉取会自动提交本地库，不会在暂存区
### 4.克隆操作
- git clone 网址链接 
- 克隆之后别名都帮你起好了   origin

### 5.删除关联的远程地址
- git remote remove 远程仓库地址别名或者网址链接
### 6.更换远程仓库的地址
- git remote set-url 旧名字 新仓库地址
### 6.如果关联了多个远程仓库
- 可以使用xrags格式（自行学习）
- 这个可以把一个分支推送到多个远程仓库
- git push --all  别名或者网址链接
- 把所有分支推送到一个远程仓库上
